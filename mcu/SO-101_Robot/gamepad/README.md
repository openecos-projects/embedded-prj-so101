# PS2 手柄驱动说明

## 概述

本驱动实现了 PS2 无线手柄接收器的通信协议，使用 GPIO 软件模拟 SPI 主机模式。

## 硬件连接

| 手柄接收器 | 单片机引脚 | 说明 |
|-----------|-----------|------|
| CLK | GPIO 0 | 时钟信号 |
| DAT | GPIO 1 | 数据输入（需上拉） |
| CMD | GPIO 2 | 命令输出 |
| CS | GPIO 3 | 片选信号 |
| VCC | 3.3V/5V | 电源 |
| GND | GND | 地 |

## 通信协议

- **协议类型**: SPI Mode 3 (CPOL=1, CPHA=1)
- **位序**: LSB First（低位在前）
- **时钟频率**: 约 31kHz（16us 半周期）
- **命令序列**: `0x01 0x42 0x00 0x00 0x00 0x00 0x00 0x00 0x00`
- **握手标志**: `data[2] == 0x5A` 表示通信成功

## 数据格式

接收到的 9 字节数据：
- `data[0]`: 0xFF（固定）
- `data[1]`: 0x41（数字模式）或 0x73（模拟模式）
- `data[2]`: 0x5A（握手成功标志）
- `data[3]`: 按键字节 1（低电平有效，需取反）
- `data[4]`: 按键字节 2（低电平有效，需取反）
- `data[5-8]`: 摇杆数据（仅模拟模式）

## 按键映射

### 按键字节 1 (data[3])
| 位 | 按键 |
|----|------|
| 0 | SELECT |
| 1 | L3（左摇杆按下） |
| 2 | R3（右摇杆按下） |
| 3 | START |
| 4 | UP（上） |
| 5 | RIGHT（右） |
| 6 | DOWN（下） |
| 7 | LEFT（左） |

### 按键字节 2 (data[4])
| 位 | 按键 |
|----|------|
| 0 | L2 |
| 1 | R2 |

| 3 | R1 |
| 4 | △（三角） |
| 5 | ○（圆圈） |
| 6 | ×（叉） |
| 7 | □（方块） |

## 关键技术问题

### 1. GPIO 寄存器操作

**问题**: SDK 的 `gpio_set_level()` 直接使用 `|=` 和 `&=` 操作寄存器，会导致其他位被意外清零。

**解决**: 使用读-改-写方式：
```c
uint32_t tmp = REG_GPIO_0_DR;
tmp |= (1 << pin);  // 或 tmp &= ~(1 << pin);
REG_GPIO_0_DR = tmp;
```

### 2. 幽灵时钟脉冲

**问题**: 初始化时先配置 DDR（方向寄存器）再设置 DR（数据寄存器），会在引脚切换为输出时产生低电平毛刺，导致手柄误认为收到时钟信号，造成 1-bit 数据错位。

**解决**: 先设置 DR 为高电平，再配置 DDR：
```c
// 1. 先设置输出值
REG_GPIO_0_DR |= (1 << pin);
// 2. 再配置为输出
REG_GPIO_0_DDR &= ~(1 << pin);
```

### 3. 影子寄存器

**问题**: 每次读取 DR 寄存器时，可能读到不稳定的物理电平，导致其他引脚状态被错误修改。

**解决**: 使用全局变量记住输出值，避免读取物理寄存器：
```c
static uint32_t gpio_out_shadow = 0;

void gpio_write(uint8_t pin, uint8_t val) {
    if(val) {
        gpio_out_shadow |= (1 << pin);
    } else {
        gpio_out_shadow &= ~(1 << pin);
    }
    REG_GPIO_0_DR = gpio_out_shadow;
}
```

### 4. 时序要求

- **CLK 半周期**: 16us
- **CS 拉低延时**: 50us
- **CS 拉高延时**: 100us
- **采样时机**: CLK 下降沿后 16us

## API 使用

### 初始化
```c
ps2_init();
```

### 读取手柄数据
```c
uint8_t data[9];
if(ps2_read_packet(data)) {
    // 握手成功
    uint8_t btn1 = ~data[3];  // 取反得到按键状态
    uint8_t btn2 = ~data[4];

    if(btn1 & 0x10) {
        // UP 键按下
    }
}
```

## 调试技巧

1. **检查握手**: `data[2]` 应该等于 90 (0x5A)
2. **检查按键**: 不按键时 `data[3]` 和 `data[4]` 应该是 255
3. **万用表测试**: 初始化后所有输出引脚应该是 3.3V
4. **数据错位**: 如果数据偏移 1 位，检查初始化顺序和影子寄存器

## 常见问题

**Q: 接收全是 255**
- 检查连线是否正确
- 检查手柄是否开机并配对
- 检查 DAT 引脚是否配置上拉

**Q: 数据错位（如收到 57, 173 而不是 65, 90）**
- 检查是否使用了影子寄存器
- 检查初始化顺序（先 DR 后 DDR）

**Q: 只有第一次有数据，之后全是 255**
- 检查 CS 片选是否正确释放
- 增加 CS 拉高后的延时
